# 图片预览与双击行为优化方案

## 问题分析

### 问题 1：图片格式加载体验差
- **现状**：双击图片 → 弹出全屏预览 → 调用 `getImageDataUrl()` 加载原图 base64 → 显示 loading spinner → 加载完成显示
- **问题**：原图可能几 MB 甚至十几 MB，base64 编码后更大，加载慢、内存占用高、用户等待时间长

### 问题 2：图片预览缺少关闭按钮（不明显）
- **现状**：工具栏最右侧有一个 `X` 按钮（`.preview-close-btn`），但不够显眼
- **问题**：用户反馈"没有关闭预览按钮"，实际上有但不够醒目；同时缺少 ESC 键和点击背景关闭的明确提示

### 问题 3：双击图片行为不可配置
- **现状**：双击图片 → 硬编码为"打开预览"；双击文本 → 可配置为"复制"或"粘贴"
- **问题**：用户可能期望双击图片直接复制（快速转发图片），而不是打开预览。应该像文本一样支持配置

---

## 解决方案

### 修改 1：图片加载优化 — 渐进式加载 + 缩略图过渡

**思路**：利用卡片列表已有的缩略图，先显示缩略图（秒开），再异步加载原图替换。

**文件**：`src/components/CardList.tsx`

#### 1.1 新增缩略图过渡状态

```typescript
// 在 openImagePreview 中：
// 先尝试获取已缓存的缩略图作为占位
const thumbnailUrl = await getImageThumbnail(item.content).catch(() => "");
setPreviewImage(thumbnailUrl || null);  // 先显示缩略图（秒开）
setPreviewLoading(false);  // 不显示 loading，直接显示缩略图

// 后台异步加载原图
const [dataUrl, info] = await Promise.all([
  getImageDataUrl(item.content),
  getImageInfo(item.content),
]);
if (dataUrl) {
  setPreviewImage(dataUrl);  // 替换为原图（淡入过渡）
  setPreviewInfo(info);
}
```

#### 1.2 CSS 过渡效果

```css
.image-preview-img {
  transition: opacity 0.3s ease;
}
```

**效果**：双击图片 → 缩略图立即出现（0 延迟）→ 原图加载完成后平滑替换

```typescript
// 在预览显示时监听 ESC 键
useEffect(() => {
  if (!previewImage && !previewLoading) return;
  const onKeyDown = (e: KeyboardEvent) => {
    if (e.key === "Escape") closePreview();
  };
  window.addEventListener("keydown", onKeyDown);
  return () => window.removeEventListener("keydown", onKeyDown);
}, [previewImage, previewLoading, closePreview]);
```

### 修改 3：双击图片行为可配置

**思路**：在 `AppConfig` 中新增 `image_double_click_action` 字段，在设置面板增加开关。

#### 3.1 Store 类型扩展

**文件**：`src/stores/appStore.ts`

```typescript
export interface AppConfig {
  // ... 现有字段
  double_click_action: "copy" | "paste";       // 双击文本行为
  image_double_click_action: "preview" | "copy"; // 双击图片行为（新增）
}
```

默认值：
```typescript
const DEFAULT_CONFIG: AppConfig = {
  // ...
  double_click_action: "copy",
  image_double_click_action: "preview", // 默认：打开预览
};
```

#### 3.2 双击逻辑修改

**文件**：`src/components/CardList.tsx`

```typescript
const handleDoubleClick = useCallback(async (item: HistoryItem) => {
  if (item.type === "image" && item.content) {
    const action = useAppStore.getState().config.image_double_click_action || "preview";
    if (action === "copy") {
      // 复制图片到剪贴板
      setPastingId(item.id);
      try {
        const dataUrl = await getImageDataUrl(item.content);
        const resp = await fetch(dataUrl);
        const blob = await resp.blob();
        await navigator.clipboard.write([new ClipboardItem({ [blob.type]: blob })]);
        toast("图片已复制", "success");
      } catch {
        toast("复制图片失败", "error");
      } finally {
        setPastingId(null);
      }
    } else {
      openImagePreview(item);
    }
  } else if (item.type === "file") {
    setFileDetailItem(item);
  } else if (item.type === "text") {
    // ... 现有逻辑不变
  }
}, [openImagePreview, toast]);
```

#### 3.3 设置面板新增开关

## 交互对比

### 修改前

| 场景 | 行为 |
|------|------|
| 双击图片 | 弹出 loading → 等待原图 → 显示 |
| 关闭预览 | 工具栏 X（不明显）、点击背景 |
| 键盘关闭 | ❌ 无 |
| 双击图片复制 | ❌ 不支持 |

### 修改后

| 场景 | 行为 |
|------|------|
| 双击图片（预览模式） | 缩略图秒开 → 原图平滑替换 |
| 双击图片（复制模式） | 直接复制到剪贴板 + Toast |
| 关闭预览 | ESC / 点击背景 / 右下角大按钮 / 工具栏 X |
| 操作提示 | 底部显示快捷键说明 |

---

## 优先级

| 修改 | 优先级 | 理由 |
|------|:-----:|------|
| 2. ESC + 关闭按钮 | **P0** | 用户直接反馈，影响基本可用性 |
| 1. 渐进式图片加载 | **P1** | 大图加载体验差，缩略图已存在可直接复用 |
| 3. 双击行为可配置 | **P1** | 用户明确需求，改动小、收益高 |
