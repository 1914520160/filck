## 二、具体问题分类

### 问题 A：Footer 不统一（5 个弹窗）

| 弹窗 | 当前 Footer | 问题 |
|------|-----------|------|
| **HelpDialog** | `.h-footer`（全宽大按钮 "我知道了"） | 不使用 `dialog-footer`，样式独立 |
| **SettingsDialog** | `.s-footer`（全宽大按钮 "关闭设置"） | 不使用 `dialog-footer`，样式独立 |
| **AboutDialog** | 无 Footer | 弹窗只有 body 内容，无底部操作区 |
| **ExtractDialog** | 无 Footer | 弹窗只有 body 内容，无底部操作区 |
| **SnippetsDialog** | 无 Footer | 弹窗只有 body 内容，无底部操作区 |

### 问题 B：硬编码颜色（4 个弹窗 + 大量 CSS）

**ExtractDialog / SnippetsDialog 组件内联样式**（已使用部分 CSS 变量）：
- `ExtractDialog`：类型选择器、结果列表、复选框等大量使用内联 `var(--xxx)` 已基本合规
- `SnippetsDialog`：片段卡片、编辑表单、搜索框基本使用 CSS 变量

**app.css 中仍有硬编码颜色**（影响所有弹窗）：

| CSS 选择器 | 硬编码值 | 应改为 |
|-----------|---------|-------|
| `.dialog-close` | `color: #86868B` | `var(--text-muted)` |
| `.dialog-close:hover` | `background: #F4F4F5` | `var(--section-bg)` |
| `.btn-primary` | `background: #0078D4` | `var(--accent)` |
| `.s-toggle.on` | `background: #34C759` | `var(--toggle-on-bg)` 或直接用 `var(--accent)` |
| `.s-theme-dot.active` | `border-color: #0078D4` | `var(--accent)` |
| `.h-feature-dot` | `background: #0078D4` | `var(--accent)` |
| `.h-key .plus` | `color: #AEAEB2` | `var(--text-ter)` |
| `.s-kbd:hover` | `background: #EFF4FF; color: #0078D4; border-color: #0078D4` | `var(--accent-light)` 等 |
| `.s-kbd.recording` | `border-color: #0078D4; background: #EFF4FF; color: #0078D4` | 同上 |
| `.s-save-btn` | `background: linear-gradient(135deg, #0078D4, #005A9E)` | 使用 `var(--accent)` 渐变 |
| `.h-close-btn` | 同上 | 同上 |
| `.s-action.danger` | `color: #EF4444` | `var(--danger)` |

### 问题 C：重复关闭按钮（4 个弹窗）

与上次图片预览同样的问题——Header 已有 X 关闭按钮，Footer 又有：
- **EditDialog**：Footer 右侧有"取消"按钮
- **FileDetailDialog**：Footer 右侧有"关闭"按钮
- **QuickPreview**：Footer 右侧有"关闭"按钮
- **HelpDialog**：Footer 全宽"我知道了"按钮（唯一操作）
- **SettingsDialog**：Footer 全宽"关闭设置"按钮（唯一操作）

### 问题 D：底部按钮文字可能换行

- **EditDialog**："💾 保存修改"（4 字 + emoji）→ 可能在某些宽度下挤压
- **FileDetailDialog**："关闭"（2 字）→ OK
- **QuickPreview**："关闭"（2 字）→ OK
- **HelpDialog**："我知道了"（4 字）→ 全宽按钮不会换行
- **SettingsDialog**："关闭设置"（4 字）→ 全宽按钮不会换行

### 问题 E：关于弹窗无 Footer

`AboutDialog` 只有 body 没有 footer，body 里有"检查更新"按钮，但缺少统一的关闭操作入口。

## 三、优化方案

### 方案总览

| 改动项 | 涉及文件 | 改动类型 |
|-------|---------|---------|
| 1. CSS 硬编码颜色 → CSS 变量 | `app.css` | 样式修正 |
| 2. HelpDialog Footer → `dialog-footer` | `HelpDialog.tsx` + `app.css` | 结构统一 |
| 3. SettingsDialog Footer → `dialog-footer` | `SettingsDialog.tsx` + `app.css` | 结构统一 |
| 4. AboutDialog 补充 Footer | `AboutDialog.tsx` | 结构统一 |
| 5. ExtractDialog 补充 Footer | `ExtractDialog.tsx` | 结构统一 |
| 6. SnippetsDialog 补充 Footer | `SnippetsDialog.tsx` | 结构统一 |
| 7. 去除重复关闭按钮 | `EditDialog.tsx`、`FileDetailDialog.tsx`、`QuickPreview.tsx` | 精简 |
| 8. 版本号更新 | `tauri.conf.json` | 版本 |

### 具体改动

#### 1. app.css — 硬编码颜色修正

替换所有硬编码颜色为 CSS 变量，涉及：
- `.dialog-close`、`.dialog-close:hover`
- `.btn-primary`
- `.s-toggle.on`、`.s-theme-dot.active`
- `.h-feature-dot`、`.h-key .plus`
- `.s-kbd:hover`、`.s-kbd.recording`
- `.s-save-btn`、`.h-close-btn`
- `.s-action.danger`

#### 2. HelpDialog — Footer 统一

**当前**：
```jsx
<div className="h-footer">
  <button className="h-close-btn">我知道了</button>
  <span className="h-ver">剪贴板管理 v{appVersion}</span>
</div>
```

**改为**：
```jsx
<div className="dialog-footer">
  <span className="h-ver">剪贴板管理 v{appVersion}</span>
  <button className="btn-primary" onClick={onClose}>我知道了</button>
</div>
```

> 与 `dialog-footer` 布局一致：左侧版本号，右侧操作按钮。可删除 `.h-footer`、`.h-close-btn` CSS。

#### 3. SettingsDialog — Footer 统一

**当前**：
```jsx
<div className="s-footer">
  <button className="s-save-btn">关闭设置</button>
  <span className="s-footer-ver">剪贴板管理 v{appVersion}</span>
</div>
```

**改为**：
```jsx
<div className="dialog-footer">
  <span className="s-footer-ver">剪贴板管理 v{appVersion}</span>
  <button className="btn-secondary" onClick={onClose}>关闭</button>
</div>
```

> 与 `dialog-footer` 布局一致。可删除 `.s-footer`、`.s-save-btn` CSS。

#### 4. AboutDialog — 补充 Footer

在 body 下方增加 `dialog-footer`：
```jsx
<div className="dialog-footer">
  <span></span>
  <button className="btn-secondary" onClick={onClose}>关闭</button>
</div>
```

Body 中的"检查更新"按钮保留，Footer 只放关闭入口。

#### 5. ExtractDialog — 补充 Footer

在 `dialog-body` 下方增加：
```jsx
<div className="dialog-footer" style={{ justifyContent: "flex-end" }}>
  <button className="btn-secondary" onClick={onClose}>关闭</button>
</div>
```

#### 6. SnippetsDialog — 补充 Footer

同上，在 `dialog-body` 下方增加统一的关闭 Footer。

#### 7. 去除重复关闭按钮

- **EditDialog**：Footer 去掉"取消"按钮，只保留"💾 保存" + 快捷键提示
- **FileDetailDialog**：Footer 去掉"关闭"按钮，只保留时间信息
- **QuickPreview**：Footer 去掉"关闭"按钮，只保留快捷键提示
- **HelpDialog**：已通过 Footer 统一解决
- **SettingsDialog**：已通过 Footer 统一解决

## 五、预期效果

1. **所有 10 个弹窗 Footer 统一**：全部使用 `dialog-footer` 类，布局一致（左信息/右操作）
2. **所有硬编码颜色消除**：支持浅色/深色主题切换
3. **无重复关闭按钮**：Header X + Footer 操作分离，不再有冗余
4. **按钮文字不换行**：使用标准 `btn-primary`/`btn-secondary`，已自带 `white-space: nowrap`
5. **版本号**：`4.5.0` → `4.6.0`
