根因分析
CSS 默认使用 content-box 盒模型。在这种模式下，width: auto（即 100%）只设定内容区宽度，padding 和 border 会额外叠加到元素总宽度上。
搜索框: padding 0 14px + border 1px
→ 溢出 = 14×2 + 1×2 = 30px

Tab控件: padding 3px
→ 溢出 = 3×2 = 6px

差距 = 30 - 6 = 24px（搜索框更宽）
当前状态 — 不对齐
搜索框比 Tab 控件宽 24px，因为 padding/border 在 content-box 模式下溢出

修复方案
在 app.css 顶部加一行全局 box-sizing: border-box，让所有元素的 padding 和 border 包含在 width 内，不再溢出。
/* 文件顶部新增 */
* { box-sizing: border-box; }

效果：
搜索框: 100% width 内含 padding + border
→ 实际宽度 = 368px（不溢出）

Tab: 100% width 内含 padding
→ 实际宽度 = 368px（不溢出）
这是现代 CSS 的标准做法，只需加一行代码，不影响已有任何样式。
